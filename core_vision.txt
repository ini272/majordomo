Project Brief: The Gamified Chore Quest System

I. The Core Vision: Re-enchanting the Mundane

The fundamental goal is to transform monotonous household chores into an engaging, fantasy-themed RPG. The project leans on the "cozy effect" of fantasy settings to combat the ambiguity and monotony of daily life by providing:

    A Sense of Purpose: Framing chores as "quests" gives them clear goals and a narrative significance.
    Moral Clarity: A chore, like a fantasy quest, has an unambiguous "good" outcome, providing a restful escape from complex real-world decisions.
    Agency and Progression: Users aren't just doing tasks; they are heroes leveling up, seeing their actions have a direct and measurable impact.

By calling the kitchen "The Forge" or taking out the trash "defeating the Trashel Elemental," the system re-enchants the home, making it a place of adventure.

II. The Technology Stack

The stack is designed to be modern, efficient, and well-integrated.

    Back-End (The Engine Room):
        Framework: FastAPI for its speed and simplicity.
        Database/ORM: SQLModel as the ideal companion to FastAPI, combining data validation (Pydantic) and database interaction (SQLAlchemy) into one.
        Database: SQLite for its simplicity and self-contained, file-based nature, perfect for self-hosting.
        Server: Uvicorn to run the FastAPI application.
    Front-End (The Quest Log):
        Framework: React (initialized with Vite for a fast development experience).
        UI Components: A customizable component library like Mantine or Chakra UI to build the interface quickly.
        Connectivity: The front-end and back-end will run as separate processes, connected via API calls. A CORS middleware in FastAPI is essential to allow them to communicate.

III. Core Features & Game Mechanics

These are the systems that will make the chore list a "game."

    The Asynchronous Scribe (AI-Generated Quests):
        Problem: Manually writing quests is time-consuming; live LLM calls are slow.
        Solution: A hybrid approach. When a new type of chore is created (e.g., "clean bathroom"), a FastAPI background task is triggered. This task calls a free-tier LLM API (Google Gemini Pro or Groq) to generate 5-10 creative descriptions and saves them to the database.
        Result: Quest creation is instantaneous for the user (a simple database read), but the content is rich, varied, and AI-generated.
    Character Progression & Economy:
        XP & Levels: Users have profiles with an XP bar and a level. Completing quests grants XP, and leveling up can unlock new in-game abilities (e.g., "Quest Veto").
        Gold & Shop: Quests also reward Gold, a currency used in a virtual shop to buy cosmetic items (app themes, avatars) or consumable "potions" (e.g., "Double XP for 1 hour").
    Time-Based Events:
        Daily Quests: A server job flags a random chore each day with bonus rewards.
        "Boss Fights": Large, daunting chores (like "clean the garage") are framed as a collaborative boss with a high HP pool. Smaller sub-tasks deal "damage" to the boss, encouraging the whole household to team up for a large reward.
    Social Features:
        Hall of Heroes: A weekly leaderboard tracking XP earned.
        Tavern Feed: A live activity feed showing recent accomplishments.

IV. The "Seamless & Magical" User Experience

These features bridge the gap between the digital app and the physical world.

    NFC Tags ("Arcane Sigils"):
        Concept: Place cheap NFC tags on or near appliances/locations (shed, washing machine, etc.).
        Function: Program each tag with a unique URL (e.g., yourapp.com/locations/shed). When a user taps their phone on the tag, it opens the web app directly to a page showing all available quests for that specific location.
        Seamlessness: Using iOS Shortcuts or Android's Tasker can make this an instant, one-tap action with no extra prompts.
    Animations ("Visual Flourish"):
        Purpose: Provide satisfying, game-like feedback.
        Tool: Framer Motion is the recommended library for React.
        Use Cases: Animate the layout as quests are re-ordered/completed, create springy physics for pop-up modals, and add elegant fade-in/out effects.
    Sound Effects ("Auditory Rewards"):
        Implementation: A reusable useSound React hook to cleanly manage loading and playing sounds.
        Function: Play a triumphant fanfare upon quest completion.
        Considerations: Sound must be triggered by user interaction (due to browser policies), and a global mute button is essential for a good user experience.
